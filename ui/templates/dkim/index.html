DKIM (DomainKeys Identified Mail) is a method of email authentication that allows recipients to verify the identity of the sender and ensure that the message has not been altered during transmission. This is achieved by using RSA keys to sign and verify email messages. Two keys are required for implementation: a private key for signing the email message, and a public key that is stored in the DNS record of the sender's domain and used by the recipient's mail server to verify the signature.

To generate DKIM keys, provide your domain, domain selector, and key size in the input fields. The domain selector allows you to use multiple DKIM records for a single domain and can be composed of any combination of alphanumeric characters.
